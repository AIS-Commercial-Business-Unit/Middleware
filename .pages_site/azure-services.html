What Runs in Azure

Both stacks are Azure workloads. This page documents which services each uses.

Both integration approaches run on Azure. Azure Kubernetes Service is the primary cost driver for either approach. The differences are in which managed services are used alongside AKS — not whether the workload is in Azure. All integration processing, message brokering, state management, security, and observability run in Azure for both stacks.

Service Comparison

Azure Service Java / Camel AIS+ Notes
Compute
Azure Kubernetes Service (AKS) ✓✓ Primary runtime for all service containers. Dominant cost driver for both stacks. Node pool sizing drives overall Azure spend.
Azure Container Registry ✓✓ Container image storage for CI/CD pipelines.
Azure Logic Apps Standard —Optional AIS+ stack can use Logic Apps built-in connectors for IBM MQ, DB2, and SFTP where a managed connector is preferred over a custom adapter. Not required — NServiceBus handles these natively.
Messaging & Events
Azure Service Bus (Premium) —✓ Primary message broker. Sagas, commands, events, dead-letter queues. Premium tier required for virtual network integration.
Azure Event Hubs / Confluent Kafka ✓— Java/Camel stack uses Confluent Kafka (available on Azure Marketplace) as the event streaming backbone. Azure Event Hubs provides a Kafka-compatible surface as an alternative.
Azure Event Grid OptionalOptional Used for Blob Storage trigger events (file batch processing) in both stacks.
Storage & State
Azure SQL Database —✓ Saga state persistence in AIS+. NServiceBus writes durable workflow state to SQL. Serverless tier available for dev/test.
Azure Cosmos DB (MongoDB API) Optional— Java/Camel stack may use MongoDB for integration state or document storage. Cosmos DB for MongoDB API provides a managed service option.
Azure Blob Storage ✓✓ File drop zones for batch processing. File trigger source for Platform.FileProcessing pattern.
API & Network
Azure API Management (APIM) ✓✓ API gateway for all inbound HTTP/REST requests. Policy enforcement, rate limiting, authentication.
Azure Virtual Network ✓✓ Private networking for AKS, Service Bus Premium, SQL, and Key Vault. Required for enterprise security posture.
Private Endpoints ✓✓ Used across all PaaS services to prevent public internet exposure.
Security & Identity
Azure Key Vault ✓✓ Secrets, connection strings, certificates. Replaces BizTalk SSO as runtime configuration store.
Azure App Configuration ✓✓ Feature flags, runtime configuration values. Replaces BizTalk SSO config store pattern.
Entra ID Managed Identities ✓✓ Workload identity for service-to-service authentication. No stored credentials.
Microsoft Defender for Cloud ✓✓ Security posture baseline. Applies to the Azure subscription regardless of application stack.
Observability
Application Insights ✓✓ Distributed tracing, performance metrics, failure analysis. Both stacks instrument via OpenTelemetry or direct SDK.
Azure Monitor + Log Analytics ✓✓ Centralized log aggregation, alerting, dashboards. Infrastructure metrics from AKS and all PaaS services.
NServiceBus ServiceControl / ServicePulse —✓ AIS+ only. Provides message-level monitoring: error queue management, SLA breach alerts, message audit, conversation tracing, endpoint health. Runs as a container in AKS. No equivalent in the Java/Camel stack without custom implementation.
CI/CD & IaC
GitHub Actions ✓✓ Build, test, deploy pipelines. Both stacks use GitHub Actions.
Bicep / Terraform (IaC) ✓✓ Infrastructure as code. Environment provisioning from a single pipeline run.
