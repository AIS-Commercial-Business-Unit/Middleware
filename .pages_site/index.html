Middleware Modernization — Proof of Concept

UC4 · PRS Appraisal Query Services — scatter-gather fan-out and content-based routing with mainframe chunk aggregation, running end-to-end.

UC4 — What Was Built

Two PRS workflows that replace the highest-risk BizTalk patterns in the estate. Both run against stub adapters that mirror the real IBM MQ and @Work interfaces.

WorkflowPatternWhat It Demonstrates
GetAppraisalList Scatter-Gather Fan-out to @Work (SQL) and DEIPDE07 (IBM MQ) in parallel. Merge, deduplicate, return partial result on MQ timeout.
GetAppraisalDocument Content-Based Routing + Chunk Aggregation Route by DocumentKey format. RiskID keys → WCF. Numeric keys → DEIPDE07 MQ, reassemble 64-byte base64 chunks, strip EBCDIC.

Functional Correctness

End-to-end execution against representative policy numbers. Outputs reconciled against expected BizTalk behaviour.

Observability

Every step emits a structured log entry. The EDA Flow Tracer renders a sequence diagram per request. No debugger required.

Partial Failure Handling

MQ timeout returns available results with partialResult=true rather than throwing an error. Chunk aggregation timeout aborts cleanly.

Adapter Isolation

Stubs replace the real IBM MQ and @Work adapters. Swapping to production adapters requires no route changes.
