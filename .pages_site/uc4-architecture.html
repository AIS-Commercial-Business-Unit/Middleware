Azure + Kubernetes UC4 Architecture

Integrated cloud diagram: icons, boundaries, workload identity, and traffic flows in one architecture canvas.

UC4 List: APIM → AKS Ingress → uc4-query → prs-appraisal → parallel fan-out to @Work SQL + DEIPDE07 MQ → aggregate → return via APIM.
UC4 Document: Route by document key to RiskID or MQ chunk path, aggregate from Blob Storage, persist state to Cosmos DB, return via APIM.
Workload Identity: K8s ServiceAccount → federated OIDC token → Entra Managed Identity → Key Vault / App Config access. No secrets in pods.
NServiceBus: policy-issuance uses SQL Server as saga persistence and transport. Long-running workflows survive pod restarts.
Observability: All services → OTel Collector → Azure Monitor + Application Insights. In-cluster: Grafana dashboards + Prometheus + Loki + Tempo.
