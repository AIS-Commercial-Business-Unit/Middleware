Stack Comparison

Two integration approaches evaluated against the same criteria. Criteria are weighted by relevance to this migration.

Java / Apache Camel Solution

A Spring Boot application using Apache Camel with YAML DSL for route definitions and Groovy for bean and validation logic. Routes, REST endpoints, and EIP patterns can be defined in YAML configuration files. Confluent Kafka serves as the event streaming backbone. The stack runs in containers on Azure Kubernetes Service.

Apache Camel provides native connectors for IBM MQ, DB2, and MSMQ through its JMS and JDBC components, using IBM's own client libraries. Integration with legacy systems is handled at the application layer.

The stack does not include a built-in mechanism for durable long-running workflow state. The Saga EIP is available in Camel but its default implementation (in-memory) is explicitly not recommended for production by the Apache Camel project. A production-grade implementation requires an external LRA (Long Running Action) coordinator.

AIS+ (.NET / NServiceBus)

A .NET 8 application using NServiceBus on Azure Service Bus as the messaging backbone. Domain services run as containers on Azure Kubernetes Service. NServiceBus provides Saga support with SQL Server-backed durable state persistence — state survives JVM/process crashes by design.

IBM MQ is supported as a first-class NServiceBus transport (from Particular Software). MSMQ is also a supported transport. DB2 connectivity uses IBM's Net.IBM.Data.Db2 NuGet package, which targets .NET 6 through .NET 10 and runs on Linux containers. Azure Logic Apps provides an additional optional path for IBM MQ and DB2 where a connector is preferred over a custom adapter.

NServiceBus adds a licensing cost (~$1K/year) and requires approximately 2 FTE-years of adoption investment. It has a narrower developer community than Apache Camel but deeper enterprise messaging capabilities.
